# DBA-Growth-Roadmap
✨ 从入门到资深：MySQL/Doris/分布式数据库 DBA 成长全记录 ✨  
记录我的 DBA 学习、实践、踩坑与进阶之路，目标是成为一名懂原理、能实战、会优化的资深数据库工程师。

## 📖 仓库定位
- **核心目标**：从零基础/入门级 DBA 逐步进阶为资深 DBA，覆盖 MySQL 核心原理、性能优化、高可用架构、OLAP 数据库（Doris）、分布式数据库等核心技能；
- **内容形式**：学习笔记、实战案例、问题排查记录、工具脚本、面试复盘、技术选型思考；
- **适用人群**：DBA 初学者、运维转 DBA、想进阶的中级 DBA。

## 🛣️ DBA 成长路径（分阶段）
### 阶段1：入门筑基（0-3个月）—— 掌握数据库基础与日常运维
#### 核心目标
能独立完成数据库安装、配置、日常巡检、简单故障处理。
#### 必学内容
- [ ] 数据库基础：SQL 语法（DML/DQL/DDL）、索引原理（B+树）、事务 ACID、隔离级别；
- [ ] MySQL 基础运维：
  - 安装部署（单机/多实例）、配置文件（my.cnf）核心参数调优；
  - 账号权限管理、日志（慢查询/错误/二进制日志）查看；
  - 常用命令（mysqldump/xtrabackup 备份恢复）；
- [ ] 操作系统基础：Linux 命令（top/ps/df/ss）、磁盘/内存/网络监控；
#### 实践任务
- 搭建 MySQL 单机实例，完成基础配置；
- 编写简单的数据库备份脚本（每日全量+增量）；
- 用 `explain` 分析并优化 5 条慢 SQL。

### 阶段2：进阶提升（3-12个月）—— 掌握性能优化与高可用
#### 核心目标
能独立做性能优化、搭建高可用架构、处理常见故障（主从延迟/死锁）。
#### 必学内容
- [ ] MySQL 进阶原理：
  - InnoDB 存储引擎（缓冲池/redo log/undo log/MVCC）；
  - 锁机制（行锁/表锁/间隙锁）、死锁排查与解决；
  - 主从复制（异步/半同步/GTID）原理与部署；
- [ ] 性能优化：
  - 索引优化（最左前缀/覆盖索引/避免索引失效）；
  - 慢查询分析（pt-query-digest）、数据库瓶颈定位（CPU/IO/内存）；
  - 参数调优（连接数/缓存/IO 模型）；
- [ ] 高可用架构：
  - 主从切换、MGR 集群搭建；
  - 分库分表（Sharding-JDBC）、读写分离；
#### 实践任务
- 搭建 MySQL MGR 高可用集群，模拟主库宕机故障转移；
- 优化千万级数据表的查询性能（耗时从秒级降到毫秒级）；
- 排查并解决生产环境的主从延迟/死锁问题（记录排查过程）。

### 阶段3：资深突破（1-3年）—— 掌握分布式/OLAP 与架构设计
#### 核心目标
能设计数据库架构、处理大规模集群、对接数据仓库/OLAP 场景。
#### 必学内容
- [ ] 分布式数据库：TiDB/ClickHouse 核心原理与运维；
- [ ] OLAP 数据库：Doris/StarRocks 部署、数据同步、报表优化；
- [ ] 架构设计：
  - 数据库容量规划、灾备方案（异地多活）；
  - 数据同步（CDC/Canal）、数仓对接；
  - 云原生数据库（PolarDB/CynosDB）运维；
- [ ] 自动化运维：
  - 用 Python/Shell 编写运维工具（批量巡检/故障自动告警）；
  - 容器化部署（Docker/K8s 部署数据库）；
#### 实践任务
- 搭建「MySQL → Canal → Doris」数据同步链路，支撑亿级数据报表查询；
- 设计中型业务的数据库架构（高可用+分库分表+灾备）；
- 编写自动化运维脚本（如批量修改多实例配置、慢查询自动告警）。

### 阶段4：专家沉淀（3年+）—— 技术体系化与跨领域融合
#### 核心目标
形成自己的技术体系，能解决复杂问题、主导数据库架构升级。
#### 必学内容
- [ ] 数据库内核：InnoDB 源码解读、自定义参数调优；
- [ ] 跨领域融合：数据库与大数据/云原生/AI 结合；
- [ ] 技术管理：方案评审、故障复盘、团队技术建设；
#### 实践任务
- 主导大型业务的数据库架构升级（如从 MySQL 迁移到 TiDB）；
- 输出技术分享/博客，沉淀自己的方法论；
- 建立数据库全链路监控与故障自愈体系。
